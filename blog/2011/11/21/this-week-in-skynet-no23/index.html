This Week in Skynet #22

Nov 21st, 2011

This week has been all about integrating Skynet with Zanata.

Integration with Zanata

First, the concept of a Snapshot has been added to Skynet. A Snapshot captures the state of a collection of topics that are returned by a filtered search. It is intended to be used to save significant milestones or releases, such as beta releases or GA releases.

To create a snapshot you first select the tags and field filter options that would return the topics that represent a significant release. You then select the Snapshots tab, enter a new for the new Snapshot, and click Save.

Once a Snapshot has been created, it can not be modified or deleted. So make sure that all the topics are OK before creating a Snapshot, because if any topics have invalid XML they can not be translated or included in the Docbook output, and the XML can not be changed.

When the Snapshot has been created, you can push the Topics to Zanata using the Push All To Zanata button. Zanata does not understand the concept of a Topic, so as a work around a Zanata project called Skynet is created, and each revision of a topic is uploaded as a separate file.

When a translation has been completed, click the Pull All From Zanata button. This will pull down the translated strings, and apply them to the Snapshot topics.

The translated version of the topics can then be previewed using the View Topic button.

There is still a lot lot be done (including dealing with injection points, building a Docbook ZIP file from a Snapshot, prioritising topics for translation to name a few), but as a proof of concept this work paves the way for further integration between Skynet and Zanata.

For the technically inclined, the zanatarest.jar file contains the Java classes required to use the Zanata REST interface directly via a RESTeasy client.
